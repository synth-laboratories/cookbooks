```python
from synth_ai.task.server import ProxyConfig, RubricBundle, TaskAppConfig
from synth_ai.task.tracing_utils import build_tracer_factory
from synth_ai.tracing_v3.session_tracer import SessionTracer

from my_math_app import (
    build_dataset,
    _base_task_info,
    describe_taskset,
    provide_task_instances,
    rollout_executor,
    OUTCOME_RUBRIC,
    EVENTS_RUBRIC,
    math_router,
)

def build_config() -> TaskAppConfig:
    registry, dataset = build_dataset()
    base_info = _base_task_info()
    tracer_factory = build_tracer_factory(SessionTracer, enabled=True)

    app_state = {"math_dataset": dataset}
    if tracer_factory:
        app_state["session_tracer_factory"] = tracer_factory

    proxy = ProxyConfig(enable_openai=True, enable_groq=True)
    return TaskAppConfig(
        app_id="math-single-step",
        name="Math Single Step Task",
        base_task_info=base_info,
        describe_taskset=lambda: describe_taskset(dataset),
        provide_task_instances=lambda seeds: provide_task_instances(dataset, seeds),
        rollout=rollout_executor,
        dataset_registry=registry,
        rubrics=RubricBundle(outcome=OUTCOME_RUBRIC, events=EVENTS_RUBRIC),
        proxy=proxy,
        routers=(math_router,),
        app_state=app_state,
    )
```

### 1. How OpenAI runs its Cookbook

At a high level, OpenAI separates:

* **Core docs** ‚Äî API reference & conceptual guides at `platform.openai.com`
* **Cookbook** ‚Äî a separate ‚Äúexamples & guides‚Äù property (`cookbook.openai.com`) backed by its own repo. ([OpenAI Cookbook][1])

Concretely:

1. **Separate repo for examples**

   * `openai/openai-cookbook` is its own repo. ([GitHub][2])
   * It has:

     * `articles/` ‚Äî narrative guides (Markdown/MDX)
     * `examples/` ‚Äî runnable scripts / notebooks
     * `images/` ‚Äî supporting assets
     * `registry.yaml` / `authors.yaml` ‚Äî metadata for the site (titles, tags, slugs, authors). ([GitHub][2])

2. **Site is generated from the repo**

   * `cookbook.openai.com` is a separate site with topic filters (‚ÄúAgents, Multimodal, Text, Guardrails, Optimization‚Ä¶‚Äù) that are essentially views over that registry. ([OpenAI Cookbook][1])
   * A static-site build step:

     * Reads `registry.yaml`
     * Maps each `articles/*.md` file to a route
     * Uses metadata (tags / topics) for the ‚ÄúTopics‚Äù, ‚ÄúFeatured‚Äù, ‚ÄúPopular‚Äù lists.

3. **Docs ‚ÜîÔ∏è Cookbook relationship**

   * Core docs link **out** to cookbook guides (‚ÄúSee examples in the Cookbook‚Äù) for deeper, runnable examples.
   * Cookbook pages link **back** to API docs and reference (‚ÄúRead the full API docs‚Äù).

4. **Patterns inside each cookbook page**

   Typical page structure:

   * Short ‚ÄúWhat you‚Äôll build‚Äù intro
   * Prereqs (API keys, installation)
   * Step-by-step narrative, each step with a **small code block**
   * A ‚Äúfull script / notebook‚Äù link: `View full example on GitHub` (under `examples/`).
   * Sometimes multi-language blocks (`python`, `curl`, etc.).

You don‚Äôt need to replicate the entire infra (registry, custom site) to mimic the *experience*‚Äîyou mostly need:

* A dedicated examples repo
* A clean way to surface them in docs with:

  * Topic grouping
  * Step-by-step narratives
  * ‚ÄúView full example‚Äù links
